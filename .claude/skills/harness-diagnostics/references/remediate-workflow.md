# Remediate 모드 워크플로우

> 진단 결과를 기반으로 자동 개선을 수행하고 재진단하는 closed feedback loop.
> Read-only 원칙의 예외: 명시적 사용자 동의 후에만 파일 수정을 수행한다.

---

## 전제 조건

- Audit 모드를 먼저 실행하여 현재 점수를 확보해야 한다.
- 사용자가 Remediate 모드를 명시적으로 요청해야 한다.
- 자동 적용 범위는 계약(contract)으로 합의한다.

---

## Phase 1: 현황 확인

### Audit 실행

- Audit 워크플로우(`references/audit-workflow.md`)에 따라 현재 상태를 평가한다.
- 12원칙 점수, 4차원 점수, 종합 점수/등급을 확보한다.

### 약점 식별

- 가장 낮은 점수의 원칙 상위 3개를 식별한다.
- 각 약점 원칙에 대해 `references/fix-catalog.md`에서 해당 점수 구간의 액션을 조회한다.
- Auto/Semi/Manual 분류를 확인한다.

---

## Phase 2: 계약 협상 (Sprint Contract)

`references/contract-template.md`의 협상 프로토콜에 따라 진행한다.

### 필수 합의 항목

1. **목표 점수/등급**: 도달하려는 종합 점수와 등급
2. **집중 원칙**: 우선 개선할 원칙 목록
3. **자동 적용 범위**: 에이전트가 직접 수행할 수 있는 작업 유형
4. **제외 사항**: 사용자가 직접 판단/수행할 항목
5. **최대 반복**: 반복 횟수 상한 (기본 3회, 최대 5회)

### 합의 실패 시

사용자가 목표를 설정하지 않거나 자동 적용을 허용하지 않으면, Audit 리포트를 제공하고 Remediate를 종료한다.

---

## Phase 3: 개선 실행

### Auto 액션 적용

1. fix-catalog에서 Auto 항목을 점수 영향력 순으로 정렬한다.
2. 각 액션을 순차 적용한다.
3. 적용 내역을 기록한다.

### Semi 액션 처리

1. 초안을 생성하여 사용자에게 제시한다.
2. 사용자 승인 후 적용한다.
3. 거부된 항목은 건너뛴다.

### 적용 원칙

- 한 번에 하나의 원칙에 집중한다.
- 각 액션 적용 후 해당 파일의 무결성을 확인한다.
- 기존 코드/설정을 삭제하지 않는다 — 추가/보완만 수행한다.
- 불확실한 경우 사용자에게 확인한다.

---

## Phase 4: 재진단 (Re-Audit)

### 재평가 실행

- Phase 1과 동일한 Audit 워크플로우를 재실행한다.
- 변경된 원칙의 점수를 재산출한다.
- 종합 점수/등급을 갱신한다.

### 반복 판정

| 조건 | 다음 단계 |
|------|-----------|
| 목표 점수 달성 | Phase 5 (결과 보고) — `achieved` |
| 목표 미달 + 반복 여유 있음 | Phase 3로 복귀 |
| 최대 반복 도달 | Phase 5 (결과 보고) — `max-iterations` |
| 사용자 중단 요청 | Phase 5 (결과 보고) — `user-stopped` |
| 점수 변화 없음 (plateau) | Phase 5 (결과 보고) — 추가 개선은 Manual 영역 |

### Loop State 갱신

`.harness/loop-state.json`에 현재 라운드의 점수, 적용 액션, delta를 기록한다.

---

## Phase 5: 결과 보고

### 결과 리포트

`references/contract-template.md`의 결과 리포트 형식에 따라:

- 시작/종료 점수 비교
- 원칙별 변화 테이블
- 적용된 액션 목록
- 남은 과제 (Manual 항목, 미달 원칙)

### 후속 권장

- 다음 Remediate 세션 권장 시점
- Manual 항목 처리 후 재진단 권장
- Maintenance 모드로 drift 모니터링 전환

---

## 워크플로우 요약

```
[사용자 요청] → Phase 1: Audit
                    ↓
              Phase 2: 계약 협상
                    ↓
              Phase 3: 개선 실행 ←──┐
                    ↓               │
              Phase 4: 재진단       │
                    ↓               │
              목표 달성? ──No──────┘
                    │
                   Yes
                    ↓
              Phase 5: 결과 보고
```

---

## 안전 장치

1. **Read-only 기본**: Remediate는 명시적 요청 없이 활성화되지 않는다.
2. **계약 필수**: 자동 적용 전 반드시 범위를 합의한다.
3. **최대 반복 제한**: 무한 루프를 방지한다 (최대 5회).
4. **삭제 금지**: 기존 파일/코드를 삭제하지 않는다.
5. **plateau 감지**: 점수 변화가 없으면 자동 종료한다.
6. **사용자 중단**: 언제든 중단 가능하다.
