# 📚 디자인 시스템 지식 저장소 (Docs README)

이 디렉토리는 디자인 시스템의 모든 의사결정과 구현 계획을 기록하는 '기록 시스템(System of Record)'입니다.

## 📂 디렉토리 구조 및 용도

- **/adr/** : **디자인/아키텍처 결정 기록 (Decision Records)**
  - 새로운 기술, 라이브러리, 또는 디자인 시스템의 글로벌 컨벤션을 정의할 때 작성합니다.
  - "왜 이 결정을 내렸는지"에 대한 맥락과 트레이드오프를 기록하여 시스템의 정당성을 제공합니다.

- **/exec-plans/** : **작업 실행 계획 (Execution Plans)**
  - 새로운 컴포넌트 추가나 파운데이션의 대규모 변경 전에 수립하는 계약서입니다.
  - 작업 목표, 구현 방법, 검증 기준을 사전에 합의합니다.

## ✍️ 운영 규칙
- 파일명 형식: `adr/NNN-title.md`, `exec-plans/YYYYMMDD-task-name.md`
- 실제 코드나 디자인과 맞지 않는 문서는 즉시 업데이트하거나 폐기하세요.
